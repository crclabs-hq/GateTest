## Quick start

```bash
# 1. Get an API key from gatetest.ai/account (Phase 2: register an OAuth app)
export GATETEST_API_KEY=gt_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 2. Verify the platform is reachable (no auth needed)
curl https://gatetest.ai/api/v1/health
# → { "status": "ok", "version": "v1", "timestamp": "..." }

# 3. Start a scan
curl -X POST https://gatetest.ai/api/v1/scans \
  -H "Authorization: Bearer $GATETEST_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://customer-site.example",
    "suite": "web",
    "callbackUrl": "https://your-app.example/gatetest-webhook"
  }'
# → 201 Created
# → { "id": "scn_abc123", "status": "queued", ... }

# 4. Poll for results (or wait for the callback webhook)
curl https://gatetest.ai/api/v1/scans/scn_abc123 \
  -H "Authorization: Bearer $GATETEST_API_KEY"
# → { "id": "scn_abc123", "status": "completed", "findings": [...] }
```

## Authentication

All endpoints (except `/api/v1/health`) require an API key.

Pass it in the `Authorization` header:

```
Authorization: Bearer gt_live_<32 alphanumeric>
```

OR (legacy alternative for clients that can't set Authorization):

```
X-API-Key: gt_live_<32 alphanumeric>
```

### Key prefixes

| Prefix | Mode | Anthropic spend |
|---|---|---|
| `gt_live_` | Production — real scans, real Claude calls, real billing | Real |
| `gt_test_` | Sandbox — returns canned results, free | None |

Develop your integration against `gt_test_` keys, switch to `gt_live_` for production. Same code, same endpoints, same response shapes — only the actual scan execution differs.

### Tiers

API keys are scoped to a tier that gates which suites they can run:

| Tier | Suites unlocked |
|---|---|
| `quick` | `web`, `quick`, `standard`, `wp` |
| `full` | All above + `full` (101-module deep scan) |
| `all` | Everything including `nuclear` (mutation, chaos, executive summary) |

Calling a suite outside your tier returns `403 TIER_INSUFFICIENT`.

## Endpoints

### `GET /api/v1/health`

Liveness check. **Unauthenticated.** Use this for monitoring before exchanging tokens.

**Response (200):**
```json
{
  "status": "ok",
  "version": "v1",
  "timestamp": "2026-05-15T12:34:56.789Z"
}
```

The shape is **locked** — partners use this for liveness probes; we will never break it.

### `POST /api/v1/scans`

Start a new scan.

**Request body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `url` | string | yes | The URL to scan. Must be public + http(s). Loopback / private IPs are rejected. |
| `suite` | string | no | One of `web`, `wp`, `quick`, `full`, `nuclear`. Default `web`. |
| `callbackUrl` | string | no | Where we POST the result when the scan completes. HMAC-signed. |
| `idempotencyKey` | string | no | If a duplicate is received within 24h, we return the original scan ID instead of creating a new one. |

**Response (201):**
```json
{
  "id": "scn_abc123def456",
  "status": "queued",
  "url": "https://customer-site.example",
  "suite": "web",
  "callbackUrl": "https://your-app.example/gatetest-webhook",
  "createdAt": "2026-05-15T12:34:56.789Z"
}
```

**Error responses:**

| Status | Code | Meaning |
|---|---|---|
| 400 | `BAD_REQUEST` | Invalid body (missing url, bad URL, invalid suite name) |
| 401 | `AUTH_FAILED` | Missing or invalid API key |
| 403 | `TIER_INSUFFICIENT` | Your key's tier doesn't include the requested suite |
| 429 | `RATE_LIMITED` | You've exceeded your per-hour rate limit |

### `GET /api/v1/scans/:id`

Read the status + (when complete) findings of a scan.

**Response (200) — queued or running:**
```json
{
  "id": "scn_abc123",
  "status": "queued",
  "url": "...",
  "suite": "web",
  "callbackUrl": "...",
  "createdAt": "..."
}
```

**Response (200) — completed:**
```json
{
  "id": "scn_abc123",
  "status": "completed",
  "url": "...",
  "suite": "web",
  "callbackUrl": "...",
  "createdAt": "...",
  "completedAt": "...",
  "healthScore": 73,
  "summary": {
    "errors": 5,
    "warnings": 12,
    "info": 3,
    "modulesRun": 18,
    "durationMs": 23847
  },
  "findings": [
    {
      "module": "wpExposedFiles",
      "severity": "error",
      "title": "Sensitive file exposed: wp-config.php.bak",
      "body": "Anyone on the internet can read this file...",
      "file": "wp-config.php.bak"
    }
  ]
}
```

**Response (200) — failed:**
```json
{
  "id": "scn_abc123",
  "status": "failed",
  "url": "...",
  "error": "Scan timed out fetching file tree"
}
```

**Error responses:**

| Status | Code | Meaning |
|---|---|---|
| 401 | `AUTH_FAILED` | Missing or invalid API key |
| 404 | `NOT_FOUND` | Scan ID doesn't exist OR belongs to a different API key |

## Webhooks (Phase 2 — coming next session)

When you supply `callbackUrl` on scan creation, we'll POST the result to that URL when the scan completes. Headers:

```
X-GateTest-Event:     scan.completed
X-GateTest-Delivery:  <uuid>
X-GateTest-Signature: sha256=<hmac-hex>
```

Verify the signature against the body using the secret you'll get when registering your app (Phase 2).

## Rate limits

| Tier | Requests / hour |
|---|---|
| Sandbox (test keys) | 100 |
| Standard | 1,000 |
| Partner | 10,000 |
| Enterprise | Negotiated |

Exceeded requests return `429 RATE_LIMITED` with a `Retry-After` header. Plan for backoff in your client.

## Status codes

| Code | Meaning | What to do |
|---|---|---|
| 200 | OK | Use the response body. |
| 201 | Created | Scan was queued successfully. |
| 400 | Bad Request | Check the `error` field — your body is malformed. |
| 401 | Unauthorized | Missing or wrong API key. |
| 403 | Forbidden | Key revoked OR tier insufficient. |
| 404 | Not Found | Scan ID doesn't exist or isn't yours. |
| 429 | Rate Limited | Slow down. Use exponential backoff. |
| 500 | Server Error | Our problem. Retry with backoff. |
| 503 | Service Unavailable | Briefly overloaded. Retry with backoff. |

## SDK (Phase 2)

```bash
npm install @gatetest/sdk
```

```js
import { GateTest } from '@gatetest/sdk';
const gt = new GateTest({ apiKey: process.env.GATETEST_API_KEY });
const scan = await gt.scans.create({ url: 'https://customer.example' });
const result = await gt.scans.poll(scan.id);
console.log(`Health score: ${result.healthScore}/100`);
```

Same code works in browser (with PKCE for OAuth in Phase 2), Node, edge runtimes.

## Versioning

- Endpoints live under `/api/v1/`. Breaking changes will land under `/api/v2/` — `/api/v1/` will stay supported for at least 12 months after a successor ships.
- Adding new fields to existing responses is NOT a breaking change. Your client should ignore unknown fields.
- Adding new endpoints is NOT a breaking change.
- Changing response shapes, removing fields, or changing auth requirements IS a breaking change — those go to `/api/v2/`.

## OAuth 2.0 (Phase 2 — preview)

The Phase 1 API uses API keys for simplicity. Phase 2 adds full OAuth 2.0 for partners building multi-user products.

```
1. Register your app:    https://gatetest.ai/developers/apps/new
2. OAuth flow:           https://gatetest.ai/oauth/authorize?...
3. Exchange code:        POST https://gatetest.ai/api/v1/oauth/token
4. Use access_token:     same Bearer auth as API keys
```

See `docs/integration-platform-2026-plan.md` for the full architecture.
